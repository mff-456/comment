什么是Vue？

Vue.js（读音 /vjuː/, 类似于 view）是一个构建数据驱动的 web 界面的渐进式框架。Vue.js 的目标是通过尽可能简单的 API 实现响应的数据绑定和组合的视图组件。

v-if 指令(后面赋予 可以转化为布尔类型的表达式，可以有v-show的作用以外可以实现较多的功能)

  <!-- 这是在template里使用的注释 -->
<template>
	<div>
          <div v-if="type === 'A'">A</div>
          <div v-else-if="type === 'B'">B</div>
          <div v-else>not"A" or "B"</div>
     </div>
</template>
<script>
    // 这是script里的注释
    export default {
        //  存储数据
        data(){
        return{
            // 定义了v-if里判断条件的值
            type:'C',
        }
    },
 }
</script>

v-show 指令(判断前端是否显示的作用)

<template>
	<div>
          <div v-show="type">我是否可以显示，type为0则我不能显示，type为1则可以显示</div>
     </div>
</template>
<script>
    // 这是script里的注释
    export default {
        //  存储数据
        data(){
        return{
            // 定义了v-if里判断条件的值
            type:1,
        }
    },
 }
</script>

v-for 指令(进行循环遍历数据)

<template>
	<div>
         <table>
		   <tr v-for="(item,index) in tlist" :key="index">
              	<td>
                  {{ item.text }}
              	</td>
                <td>
                  {{ item.num }}
              	</td>
                <td>
                  {{ item.price }}
              	</td>
    		</tr>
         </table>
     </div>
</template>
<script>
    // 这是script里的注释
    export default {
        //  存储数据
        data(){
        return{
            // 定义了v-if里判断条件的值
            tlist:[{text:'汽车',num:1,price:10},{text:'化妆品',num:2,price:6},{text:'牛奶',num:3,price:7}]
        }
    },
 }
</script>

v-bind 指令(v-bind指令可以在其名称后面带一个参数，中间放一个冒号隔开，这个参数通常是HTML元素的特性 attribute )

<template>
	<div>
        <h1 v-bind:src="../../xxx.jpg">这里的v-bind可以简写为:</h1>
    </div>
</template>

v-on 指令(一般用作绑定事件)

<template>
	<div>
        <button v-on:click="item++">这里的v-on:click可以简写为@click,每次点击触发事件使item+1</button>
    </div>
</template>
<script>
    // 这是script里的注释
    export default {
        //  存储数据
        data(){
        return{
			item:0
    },
 }
</script>

v-model 指令(v-model在表单控件元素上创建双向数据绑定)

<input type="text" v-model="item">
<!-- 这里的v-model绑定了输入框数据与后端数据，所以称为双向数据绑定 -->
