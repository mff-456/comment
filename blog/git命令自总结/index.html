克隆云端项目

# 新建文件夹

$ mkdir <name>
# 查看所有目录

$ dir
# 进入xx文件夹

$ cd <name>
# 克隆远程仓库的项目

$ git clone <项目网址>
# 进入克隆来的项目

$ cd <cloneproject>
# dir查看一下

# 拉取代码 (clone本地知仓库的时候被clone的远端仓库道默认被称为 origin。 所以如果你想向/从这个远端仓库push/pull 的时候，用内 origin 指代这个远端仓库。)

$ git pull origin master

上传本地项目

# 连接到云端

$ git config --global user.name "马菲菲"
$ git config --global user.email "2980738482@qq.com"
# cd (上传文件里)

# 创建git文件

$ git init
# 暂存到云端

$ git add -A
# 把暂存区中的文件提交到本地仓库中并添加描述信息

$ git commit -m '描述信息'
# 映射到云端的仓库

$ git remote add origin <云端仓库链接>
# 上传

$ git push -u origin master

修改已上传云端的项目

# cd <已上传并修改的项目>

# 连接到云端

$ git config --global user.name "马菲菲"
$ git config --global user.email "2980738482@qq.com"
# 暂存到云端

$ git add -A
# 把暂存区中的文件提交到本地仓库中并添加描述信息

$ git commit -m '修改后的vue文件'
# 上传已修改项目

$ git push origin master

git报错：‘fatal:remote origin already exists'怎么处理？附上git常用操作以及说明

只要两步：

1、先删除

$ git remote rm origin

2、再次执行添加就可以了。

git查看远程仓库地址命令

$ git remote -v
